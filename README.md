# IFA — официальный статический сайт

Готовая версия сайта Международной Федерации Автотуризма для Vercel. Интерфейс остаётся статическим и не требует сборки, а серверная функция Vercel безопасно записывает заявки в закрытый Google-реестр.

## Структура

- `index.html` — главная страница;
- `about.html` — страница «О нас»;
- `idp-department.html` — Департамент выдачи Международного Водительского Удостоверения;
- `cooperation.html` — страница сотрудничества;
- `material-*.html` — материалы экспертного центра;
- `styles.css`, `about.css`, `idp-department.css`, `cooperation.css`, `material.css` — стили;
- `app.js`, `navigation.js` — языки, меню, карусели, анимации и форма;
- `api/partnership.js` — серверная приёмка и запись заявок в Google Sheets;
- `assets/` — только изображения, которые используются сайтом;
- `robots.txt`, `sitemap.xml` — правила индексации и карта публичных страниц;
- `.env.example` — перечень переменных окружения без секретных значений;
- `vercel.json` — настройки чистых URL и HTTP-заголовков для Vercel.

Изображения интерфейса оптимизированы в WebP; оригиналы-дубликаты и неиспользуемые файлы удалены. Для фотографии команды сохранён JPEG, поскольку его WebP-копия была больше исходника.

## Локальная проверка

Откройте терминал в папке проекта и запустите любой статический сервер, например:

```powershell
python -m http.server 8080
```

Затем откройте `http://localhost:8080`. При таком запуске можно проверить интерфейс, но отправка формы работает только через Vercel Function.

Для полной локальной проверки формы используйте `vercel dev` после добавления локального `.env.local`. Файл с секретами исключён из Git и не должен попадать в GitHub.

## Загрузка в GitHub через браузер

1. Создайте новый пустой репозиторий на GitHub. Не добавляйте при создании README, `.gitignore` или лицензию.
2. В репозитории нажмите **Add file → Upload files**.
3. Перетащите **содержимое этой папки**, а не саму внешнюю папку.
4. Убедитесь, что `index.html` лежит в корне репозитория, а изображения — внутри `assets`.
5. Нажмите **Commit changes**.

В GitHub должны попасть:

- папки `api/` и `assets/`;
- семь HTML-страниц: `index.html`, `about.html`, `idp-department.html`, `cooperation.html` и три файла `material-*.html`;
- пять CSS-файлов: `styles.css`, `about.css`, `idp-department.css`, `cooperation.css`, `material.css`;
- `app.js` и `navigation.js`;
- `robots.txt`, `sitemap.xml`, `vercel.json` и `README.md`;
- `.gitignore`, `.vercelignore` и `.env.example`.

Не загружайте `.git/`, `.agents/`, `.vercel/`, `.env`, `.env.local`, архивы и любые JSON-ключи Google. Эти данные либо локальные, либо секретные.

## Публикация на Vercel

1. В Vercel нажмите **Add New → Project** и импортируйте созданный GitHub-репозиторий.
2. Выберите **Framework Preset: Other**.
3. Оставьте **Root Directory** равным корню репозитория.
4. Включите переопределение **Build Command** и оставьте поле пустым. Для этого сайта сборка не нужна.
5. **Output Directory** оставьте по умолчанию (`.`).
6. Нажмите **Deploy** и проверьте выданный адрес `*.vercel.app`.

После подключения репозитория каждый новый commit в основной ветке автоматически создаёт новое production-развёртывание.

## Подключение реестра заявок Google Sheets

Рабочая таблица уже создана в папке Google Drive `IFA APPLICATIONS`:

`https://docs.google.com/spreadsheets/d/1Fcdl-czjS5v4HGLz_2QLDlkwHqyIMtg_28VUSaaLVME`

Для включения автоматической записи требуется однократная настройка доступа:

1. В Google Cloud создайте или выберите проект и включите **Google Sheets API**.
2. Откройте **IAM & Admin → Service Accounts**, создайте сервисный аккаунт и выпустите для него ключ JSON.
3. Откройте таблицу по ссылке выше, нажмите **Настройки доступа** и добавьте email сервисного аккаунта с ролью **Редактор**.
4. В Vercel откройте **Project → Settings → Environment Variables** и добавьте для Production, Preview и Development:
   - `GOOGLE_SERVICE_ACCOUNT_EMAIL` — `client_email` из JSON-ключа;
   - `GOOGLE_PRIVATE_KEY` — значение `private_key` из JSON-ключа целиком;
   - `GOOGLE_SHEETS_ID` — `1Fcdl-czjS5v4HGLz_2QLDlkwHqyIMtg_28VUSaaLVME`;
   - `GOOGLE_SHEET_NAME` — `Заявки`.
5. Повторно разверните проект и отправьте одну тестовую заявку. Она должна появиться первой свободной строкой листа `Заявки`.

Не загружайте JSON-ключ в GitHub и не вставляйте его в `app.js`. Секрет используется только серверной функцией Vercel.

## Подключение домена `ifa.org.kz`

1. Откройте проект в Vercel: **Settings → Domains**.
2. Добавьте `ifa.org.kz`, затем при необходимости `www.ifa.org.kz`.
3. Vercel покажет точные DNS-записи. Внесите именно их в панели регистратора домена.
4. Назначьте один адрес основным, а второй перенаправьте на него.
5. Дождитесь статуса **Valid Configuration** — SSL-сертификат будет выпущен автоматически.

Не копируйте DNS-значения из сторонних инструкций: используйте значения, которые Vercel показывает именно для вашего проекта.

## Как работает форма

После настройки переменных Vercel форма отправляет данные на `/api/partnership`. Сервер проверяет обязательные поля, согласие, honeypot и частоту запросов, затем присваивает номер заявки и добавляет строку в Google Sheets. Локальные копии на устройстве посетителя не создаются.
